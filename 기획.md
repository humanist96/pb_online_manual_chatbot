# PowerBASE 온라인매뉴얼 "계좌" 부문 RAG Q&A 챗봇 구축 계획

## Context (배경)

코스콤 원장시스템(PowerBASE) 온라인 매뉴얼이 `http://211.255.203.234:8080/000/ST/`에
**Adobe RoboHelp 2022**로 배포되어 있다. 이 중 **"계좌" 부문**의 HTML 토픽들을
파싱하여 사내 담당자용 **Q&A RAG 챗봇**을 만드는 것이 목표다.

사용자가 `0878_AC250400_지점계좌서비스약정등록내역.pdf` → `.xlsx`로 수작업 변환한
예시가 있으며, 이 XLSX 포맷(계층 브레드크럼 + 설명)이 RAG에 이상적임을 확인했다.
이번 단계는 **AC250400 한 화면을 파일럿**으로 삼아 파싱 파서를 완성·검증하고,
동일 파서로 계좌 섹션 전체로 확장하며, **XLSX + RAG용 JSONL**을 동시 산출한 뒤
**오픈소스·로컬 실행 RAG 챗봇**까지 구축한다(외부 상용 API 미사용).

### 결정된 요구사항 (사용자 확인 완료)
- **범위**: AC250400 파일럿 우선 → 검증 후 계좌 전체 확장
- **메타정보**: HTML/URL에서 추출 가능한 값만 자동 채움 (발행부서·버전·성명 등은 공란/플레이스홀더)
- **산출물**: 화면별 XLSX + RAG용 청크 JSONL(메타데이터 포함) 동시 생성
- **챗봇**: RAG 챗봇 본체까지 구축 (임베딩+벡터검색+응답)
- **★제약(필수)**: 챗봇 시스템 **전체를 오픈소스 기반으로, 로컬에서 실행**.
  외부 상용 API(Claude/OpenAI/Voyage 등) **일절 사용 금지** — 임베딩·리랭커·LLM 모두 로컬 모델.
  (사내 폐쇄망·데이터 외부 유출 방지 목적으로 해석)

## 소스 구조 분석 (완료)

소스: `http://211.255.203.234:8080/000/ST/AC250400.html` (23KB, 200 OK).
형제 계좌 토픽 다수 확인: `AC250200`, `ACA51000`, `ACA50400` 등 → `AC*.html` 집합.

HTML 계층 구조 (CSS class 기반):
- `div.title_box` → **대분류**: 화면알아보기 / 용어찾기 / 질문보기
- `div.Step00_icon` → **중분류**: 화면개요 / 화면설명 / 관련화면
- `div.Step1_N01`, `Step1_N02` → 화면설명 내 **단계**(조건입력/조회결과)
- `td.bground_blue` → **항목**, 대부분 `"용어 : 설명"` 패턴 (콜론 분리 대상)
- `span.icon01`(58개) / `span.icon02`(22개) → **중첩 리스트 레벨**(들여쓰기 계층)
- `div.T_QAbox` > `.Que` / `.Ans` → **Q&A 쌍** (질문보기)
- `h1 > span` → 화면 제목 `"지점계좌서비스약정등록내역[0878]"` (screen number `[0878]`)
- `meta[name=gTopicId]` → `8.10.8.0_3` (TOC 계층 위치)

목표 XLSX 포맷 (B/C 2열, 1017행, sharedStrings 119개로 확인):
- 상단 메타 블록: 제목/코드(`0878-AC250400`)/유형/발행부서/파일명/버전/작성일/화면TR/성명/일자
- `<메타정보>` / `<본문추출>` 구분자
- `AUP: AC250400`, `요약`
- **브레드크럼 경로 → 설명** 행 (예: `지점계좌서비스약정등록내역 > 화면설명 > 조건입력 > 조회구분 > 지점별`)
- 용어찾기(글로서리), 관련화면, 질문보기(Q&A) 섹션

**중요**: 메타 블록의 발행부서/버전/성명/작성일/화면TR은 HTML에 없음 → 코드·제목·화면번호·AUP만
자동 채우고 나머지는 공란/플레이스홀더 처리(사용자 확인).

## 구현 계획

디렉터리 구조(신규):
```
src/
  crawl.py        # 계좌 토픽 URL 수집·다운로드 (파일럿은 AC250400 단건)
  parse.py        # HTML → 구조화 트리 (핵심)
  to_xlsx.py      # 구조화 트리 → 샘플과 동일 포맷 XLSX
  to_chunks.py    # 구조화 트리 → RAG용 JSONL 청크
  build_index.py  # 청크 임베딩 → 벡터 인덱스
  chatbot.py      # RAG 챗봇 (검색 + Claude 응답 + citations)
data/
  html/           # 원본 HTML 캐시
  xlsx/           # 화면별 XLSX
  chunks.jsonl    # RAG 데이터셋
  index/          # 벡터 인덱스
requirements.txt
.env.example      # EMBED_MODEL, LLM_BACKEND(ollama/vllm), LLM_MODEL — 모두 로컬, API 키 불필요
```

### 1단계 — 파싱 (`src/parse.py`) ★핵심, 품질의 90%가 여기서 결정
- **BeautifulSoup4 + lxml**로 파싱. RoboHelp 런타임 JS는 무시하고 정적 HTML 토픽만 사용.
- `<body>`에서 `script`/`style`/헤더 네비 제거 후 **본문 컨테이너**만 추출.
- **계층 재구성 알고리즘**:
  1. `title_box`(대분류) → `Step00_icon`(중분류) → `Step1_Nxx`(단계) 순서로 섹션 트리 생성
  2. `icon01`/`icon02` 중첩 깊이로 리스트 아이템의 부모-자식 관계 복원
  3. `td.bground_blue` 셀 텍스트를 **첫 콜론(`:`) 기준으로 `용어`/`설명` 분리**
     (콜론 없으면 전체를 설명으로) → 브레드크럼 말단 노드로 부착
  4. `\r\n`·다중 공백 정규화(`►` 불릿은 유지), `html.unescape` 적용
  5. `T_QAbox`의 `.Que`/`.Ans`를 Q&A 쌍으로 별도 수집
- **출력**: 화면당 하나의 dict —
  `{screen_id, aup, title, screen_no, topic_id, summary, breadcrumbs:[{path:[...], text}], glossary:[...], related:[...], qa:[{q,a}]}`
- 브레드크럼 `path`의 루트는 항상 문서 제목(`지점계좌서비스약정등록내역`)으로 통일(샘플과 일치).

### 2단계 — XLSX 생성 (`src/to_xlsx.py`)
- **openpyxl**로 샘플과 동일한 B/C 2열 레이아웃 재현.
- 상단 메타 블록(제목/코드/유형/AUP 등 HTML 추출값), `<메타정보>`/`<본문추출>` 구분자,
  브레드크럼 행(B열=경로, C열=설명), 용어찾기/관련화면/질문보기 순서.
- 재사용: 샘플 `0878_AC250400_...xlsx`를 **골든 파일**로 두고 셀 단위 diff 검증.

### 3단계 — RAG 청크 (`src/to_chunks.py`)
- 브레드크럼 1개 = 청크 1개. `text` 필드 = `"경로 > ... : 설명"` 형태로 경로를 문맥에 보존.
- 각 청크 메타데이터: `{screen_id, aup, screen_no, title, section_path, source_url, chunk_type}`
  (`chunk_type` ∈ summary/description/glossary/related/qa).
- Q&A 쌍은 질문을 임베딩 대상 텍스트로, 답변을 함께 저장(질의 정합성↑).
- JSONL 1줄 = 1청크. 임베딩·citations에 바로 사용.

### 4단계 — 인덱싱 (`src/build_index.py`) ※전부 로컬 오픈소스
- **임베딩(로컬)**: **BGE-M3**(`BAAI/bge-m3`, sentence-transformers) 기본 —
  다국어·한국어 강함, 한 모델로 dense/sparse/ColBERT 동시 지원.
  - 한국어 특화 대안: `nlpai-lab/KURE-v1`(고려대) 또는 `jhgan/ko-sroberta-multitask`.
  - 최초 1회만 모델 가중치 다운로드, 이후 완전 오프라인 실행(HF 캐시).
- **벡터 저장소(로컬)**: **FAISS**(`faiss-cpu`) 기본, 의존성 최소·파일 영속화.
  규모 확장 시 로컬 **Chroma** 고려(둘 다 오픈소스·로컬).
- **하이브리드 검색**: 밀집(BGE-M3 dense) + 희소(BM25, `rank_bm25`) 결합 →
  계좌 전문용어/화면번호 정확 매칭 보강.
- **리랭커(로컬, 선택)**: `BAAI/bge-reranker-v2-m3`로 top-k 재순위 → 정확도 향상.

### 5단계 — 챗봇 (`src/chatbot.py`) ※로컬 LLM, 외부 API 없음
- 질의 → 하이브리드 검색 → (리랭크) → top-k 청크를 로컬 LLM 컨텍스트로 전달 → 답변.
- **로컬 LLM 서빙**: **Ollama**(설치·운영 간단) 기본. 대안: **vLLM**(GPU 처리량↑) / `llama.cpp`(GGUF, CPU 가능).
- **모델 후보(오픈 웨이트, 한국어)**:
  - `Qwen2.5-7B/14B-Instruct` — 한국어·지시이행 우수, 라이선스 관대.
  - `LG EXAONE-3.5-7.8B-Instruct` — 한국어 특화 오픈 모델.
  - 저사양/CPU: 7B급 GGUF 양자화(Q4) 사용.
  - `.env`로 서빙 백엔드·모델명 전환 가능하게 구성.
- **출처(citations) 자체 구현**: 각 청크에 `[S1]`,`[S2]` 식 인용마커를 붙여 프롬프트에 주입하고,
  LLM이 문장 끝에 마커를 달도록 지시 → 답변에 **출처(화면번호/섹션 경로) 부착**(외부 citations 기능 없이 로컬로 재현).
- **답변 규칙**: 검색 컨텍스트에 근거가 없으면 "매뉴얼에서 확인되지 않음"으로 답하고 추측 금지(할루시네이션 억제).
- **UI(선택)**: 로컬 실행 오픈소스 **Streamlit** 또는 **Gradio**로 간단한 챗 UI 제공(전부 로컬).
- 스트리밍 토큰 출력으로 UX 개선(Ollama/vLLM 스트리밍 API).

## 품질 향상 방안 (요청 반영)
1. **골든 파일 회귀 테스트**: 샘플 XLSX와 파서 산출물을 셀 단위로 비교하는 테스트 고정.
2. **계층 복원 정밀도**: `icon01/icon02` 깊이·`bground_blue` 콜론 분리 규칙을 파일럿에서 검증 후 확장.
3. **경로 보존 청킹**: 브레드크럼 전체 경로를 청크 텍스트에 포함 → 검색 정확도·citations 근거 강화.
4. **하이브리드 검색 + 재순위**: 밀집+BM25, `bge-reranker-v2-m3` 재순위로 계좌 전문용어 매칭 개선.
5. **출처 부착 강제**: 인용마커 기반으로 모든 답변에 출처 부착 → 신뢰성·검증가능성 확보(로컬 구현).
6. **한국어 임베딩**: 한국어 특화 로컬 모델(BGE-M3 / KURE)로 도메인 용어 임베딩 품질 확보.
7. **확장 시 TOC 열거**: RoboHelp `whxdata` TOC(또는 링크 크롤링)로 계좌 섹션 `AC*.html` 전량 자동 수집.
8. **완전 오프라인 검증**: 모델 가중치 최초 캐싱 후 네트워크 차단 상태에서 파이프라인 재실행 → 외부 의존 없음 확인.

## 검증 (End-to-End)
1. `python src/parse.py data/html/AC250400.html` → 구조화 dict 출력, 브레드크럼 수/Q&A 확인.
2. `python src/to_xlsx.py` → 생성 XLSX를 샘플 골든 파일과 diff (핵심 셀 일치).
3. `python src/to_chunks.py` → `chunks.jsonl` 행 수·메타데이터 스팟 체크.
4. `python src/build_index.py` → 인덱스 생성.
5. `python src/chatbot.py "지점계좌서비스약정등록내역 화면에서 변경사용자 항목은 어디서 확인하나요?"`
   → 샘플의 질문보기(117/118)와 부합하는 **출처 포함** 답변 확인(로컬 LLM).
6. 네트워크 차단 상태로 4~5단계 재실행 → 완전 로컬 동작 확인.
7. 파일럿 통과 후 계좌 섹션 전체로 `crawl.py` 확장 → 배치 실행.

## 핵심 기술 스택 (전부 오픈소스·로컬)
- 파싱: `beautifulsoup4`, `lxml`
- XLSX: `openpyxl`
- 임베딩: `sentence-transformers` + **BGE-M3**(대안: KURE / ko-sroberta) — 로컬
- 벡터/검색: `faiss-cpu`, `rank_bm25`, (선택) `bge-reranker-v2-m3` — 로컬
- LLM 서빙: **Ollama**(기본) / vLLM / llama.cpp — 로컬, 모델: Qwen2.5-Instruct 또는 EXAONE-3.5
- UI(선택): Streamlit 또는 Gradio — 로컬
- 외부 상용 API: **없음** (완전 오프라인 실행 가능)
