# 교육자료 제작 계획서 — KOSCOM RAG 실습 과정 (웹슬라이드)

> PB 매뉴얼 데스크(로컬 RAG 챗봇) 구축 경험을 사내에 전파하는 **실습형 교육 웹슬라이드**.
> RAG 개념은 공신력 있는 웹 문서를 수집·인용해 가르치고, 실습은 **Claude Code 기반 튜토리얼**로
> 수강생이 이 저장소의 챗봇을 데이터 전처리부터 배포까지 직접 구축한다.
> 보고서(`docs/report/`)·영상(`video/`)과 **하나의 브랜드 시스템**(KOSCOM 오렌지·Pretendard)을 승계한다.

---

## 1. 개요

| 항목 | 내용 |
|---|---|
| 목적 | 사내 개발자가 **반나절 안에 로컬 RAG 챗봇을 이해하고 직접 구축**할 수 있게 하는 실습 과정 |
| 대상 | 사내 개발자·IT 담당자 — 개발 PC에 Claude Code 사용 가능 전제 |
| 형식 | **웹슬라이드**(자립형 HTML, 폐쇄망에서 브라우저로 열람) 약 40장 — 이론 30% + 실습 70% |
| 핵심 요구 | 실습 프롬프트·셸 명령이 **복사-붙여넣기로 그대로 실행**되도록 상세화 (블록마다 복사 버튼) |
| 산출물 | `docs/edu/index.html`(+ edu.css/edu.js/images/sources.md) |
| 배포 | 사내 한정(매뉴얼 화면 캡처 포함). file:// 로 열어도 동작하는 자립형 |
| 실습 프롬프트 원칙 | 이 프로젝트 실제 구축에 쓰인 요청 패턴을 정제해 승계하거나, 단계별 **최선의 Claude Code 프롬프트를 직접 설계**. 각 프롬프트에 "왜 이렇게 쓰는가" 해설 1줄 |

## 2. 디자인 — 기존 브랜드 시스템 승계

| 요소 | 승계 원천 | 적용 |
|---|---|---|
| 컬러 토큰 | `web/styles.css :root` (정본) | 오렌지 #f5821f/#e4670a(강조 전용), 텍스트 오렌지 #c9610a, 잉크 #191c20, OK 그린 #1e8a62 |
| 다크 표지 | `docs/report` 표지 · 영상 인트로 | #0b0e13 + 오렌지 그라드바 + 스파크 심볼 SVG — 표지·파트 구분·클로징 슬라이드 |
| 본문 슬라이드 | 보고서 본문 문법 | 라이트 배경, `.kicker`(오렌지 라벨) → 헤드(오렌지 `em` 강조) → 본문/블록 → 풋터(과정명+쪽번호) |
| 컴포넌트 | `docs/report/template/print.css` | `.stat`(수치 카드)·`.pipe`(파이프라인)·`.crumb`(브레드크럼 칩)·`.boundary`(보안 밴드)·`.steps`·`.checks`를 **px·16:9로 이식** |
| 폰트 | `web/fonts/PretendardVariable.woff2` | 상대참조 임베드. 코드는 D2Coding→Cascadia 폴백 |
| 이미지 | `docs/report/images/` | ui-hero/answer/gated/map/qa.png, scene-*.png 재사용 |
| 오렌지 규율 | web 주석 명문 규칙 | 슬라이드에서도 세 곳만: 킥커 · 헤드 강조 · 프롬프트 카드 테두리 |

**슬라이드 엔진**: 외부 라이브러리 없이 CSS scroll-snap 풀스크린 섹션 + 키보드(←→·Space·Home/End) + 상단 진행바 + `T` 목차 오버레이. 코드/프롬프트 블록 hover 시 **복사 버튼**(clipboard API — `web/app.js`의 `copyText` 패턴 재사용).

## 3. 커리큘럼 (약 40장)

| 파트 | 장수 | 내용 |
|---|---|---|
| **P0 오리엔테이션** | 4 | 다크 표지 · 오늘 만들 것(완성 UI 캡처+실측 수치) · 과정 로드맵 · 실습 환경 체크(복붙 검증 명령) |
| **P1 RAG 개념** | 8 | 할루시네이션 문제 → RAG 파이프라인 → 임베딩·벡터 검색 → BM25 → 하이브리드 → 리랭커·관련도 게이트 → 청킹 전략(브레드크럼=청크 사례) → 개념 정리. **각 장 하단에 출처 각주** |
| **P2 Claude Code 기초** | 6 | 설치·시작 · CLAUDE.md(이 repo 실물) · 플랜 모드 · 서브에이전트 · 프롬프트 패턴(맥락·제약·검증) · 실습 규칙 |
| **P3 핸즈온: 챗봇 구축** | 14 | 6단계 × (목표 1장 + 실습 1~2장): ① 환경 구축 ② 데이터 전처리(크롤·파싱, AC250400) ③ 청크 생성 ④ 인덱스 빌드 ⑤ 챗봇·웹 UI ⑥ 배포(systemd/Docker) |
| **P4 심화·품질** | 5 | α 혼합 가중치 실험(QA 모드) · τ 게이트·보정 · 정밀/빠름 토글 · 품질 평가 관점 · 확장 아이디어 |
| **클로징** | 3 | 트러블슈팅 FAQ · 소스 모음 · 다크 클로징 |

**실습 장 3단 구성** (P3 전체 공통):
1. ⓐ **셸 명령** — repo 루트 기준 그대로 실행 가능 (실행 검증된 것만 수록)
2. ⓑ **Claude Code 프롬프트 카드** — 복붙용 프롬프트 + "왜 이렇게 쓰는가" 해설 1줄
3. ⓒ **기대 출력 / 확인 방법** — 성공 판정 기준(예: `make test` 5/5, `/api/meta` 청크 수)

## 4. 콘텐츠 소스 전략

- **P1·P4 개념**: 서브에이전트 3개 병렬 리서치로 공신력 소스 수집(원 논문 Lewis et al. 2020, Anthropic·AWS·Google Cloud·NVIDIA·Hugging Face 공식, FAISS/sentence-transformers/BAAI 문서, Jay Alammar류 시각화 교육 자료). **접속 검증(200) 통과분만** `docs/edu/sources.md`에 큐레이션하고 슬라이드 각주로 인용
- **P2**: Anthropic 공식 문서(Claude Code quickstart·memory·subagents·best practices)만 사용
- **P3 실습**: 이 저장소가 유일한 소스 — CLAUDE.md 파이프라인 명령, 실제 구축 시 사용된 프롬프트 패턴, README 실측 시나리오

## 5. 제작 파이프라인

```
docs/edu/
├── index.html      # 슬라이드 본체 (섹션 = 슬라이드)
├── edu.css         # 디자인 토큰 + 슬라이드 컴포넌트 (print.css 이식)
├── edu.js          # 내비·진행바·목차·복사 버튼 (~150줄, 무의존)
├── images/         # report images 재사용 사본
└── sources.md      # 공신력 소스 큐레이션 (URL·발행처·활용처)
```

빌드 단계 없음(정적 파일). 열람: 브라우저로 `docs/edu/index.html` 직접 열기 또는 임의 정적 서버.
