# 온보딩·도움말 계획 — 최초 사용 가이드 + QA 모드 설명

2026-07-09. 대상: **온라인 데모(`deploy/online/`) 전용**(순수 프런트 추가 — 서버·API 무변경). 이후 사내 webapp 이식 가능.

## 1. 문제 정의 / 목표

데모를 처음 여는 사람은 화면에 개념이 많아 어디부터 손대야 할지 막막하다.

| 혼란 지점 | 현재 | 문제 |
|---|---|---|
| 3존 레이아웃 | 좌 이력 · 중 대화 · 우 근거 | 각 영역의 역할이 설명 없이 노출 |
| 범위(브레드크럼) | 컴포저의 "범위: 전체 ▾" 칩 | 왜 좁히는지(교차 오염 방지) 모름 |
| 정밀/빠른 검색 | 컴포저 토글 | 리랭커·코사인 차이 불명 |
| **QA 계측 모드** | `Q` 키·헤더 `QA` 토글·`?qa=1` | **α·τ·top-k·dense/sparse가 전부 계측 용어 — 설명 없이는 해독 불가** |
| 모바일 | 근거·이력·QA가 슬라이드 시트로 숨음 | 발견성 매우 낮음(버튼 존재를 모름) |

목표: ① 최초 1회 **환영 가이드**로 핵심 사용법을, ② 상시 **도움말 오버레이**로 재열람을, ③ QA 진입 시 **전용 코치**로 계측 용어를 알기 쉽게 설명한다. **셋 다 모바일에서 동등하게 동작**한다.

## 2. 설계 원칙

- **무의존·자체완결 오버레이** — 기존 피드백 모달(`fb-modal`)·스크림 패턴을 재사용. 레이아웃 설명은 **인라인 SVG 미니 목업**으로 그려 데스크톱/모바일에서 동일하게 렌더(요소 위에 절대좌표로 얹는 스포트라이트 투어는 모바일에서 근거·이력이 화면 밖이라 깨지므로 **지양**).
- **강요하지 않음** — 어느 단계서든 건너뛰기·"다시 보지 않기", `Esc`·스크림 클릭으로 닫기.
- **계층 분리** — 일반 사용자는 3~4스텝이면 끝. QA 계측은 **필요할 때(첫 진입)만** 별도 코치로 — 일반 사용자에게 계측 용어를 들이밀지 않는다.
- **버전드 재노출** — localStorage 플래그에 가이드 버전을 저장, 가이드를 크게 개편하면 자동 재노출.
- **접근성** — 포커스 트랩, `aria-modal`, `Esc` 닫기, `prefers-reduced-motion` 존중, 큰 탭 타깃.

## 3. 구성 — 진입점 4곳

1. **최초 방문 환영 오버레이** — `/api/meta` 로드 후 `pbdesk.onboarded` 플래그가 없으면 자동 표시(첫인상 지연 방지 위해 메타 성공 후).
2. **헤더 「?」 도움말 버튼** — 상시. 탭형 도움말을 언제든 재열람(`fb-open` 옆에 배치).
3. **QA 첫 활성화 코치** — `Q`/토글/`?qa=1`로 QA를 처음 켤 때 `pbdesk.qacoached`가 없으면 1회 표시.
4. **빈 히어로 보조 링크** — "처음이신가요? 사용법 보기"(hero 하단, 칩 아래) — 자동 노출을 닫은 뒤에도 눈에 띄는 재진입점.

## 4. 환영 오버레이 (일반 사용자) — 4스텝 캐러셀

각 스텝은 짧은 문장 + 인라인 SVG 미니 목업. 하단 점 인디케이터 + 이전/다음/「시작하기」. 모바일은 좌우 스와이프.

```
┌─ PowerBASE 매뉴얼 데스크 ───────────── 건너뛰기 ✕ ┐
│                                                  │
│   [ SVG 미니 목업 ]                               │
│                                                  │
│   ② 물어보고, 근거로 확인하세요                    │
│   질문하면 매뉴얼 근거에서 답을 만들고,             │
│   문장의 [S1] 표시로 어디서 나왔는지 알려줘요.      │
│                                                  │
│            ● ○ ○ ○      [이전]  [다음]            │
└──────────────────────────────────────────────┘
```

| 스텝 | 제목 | 요지 | 미니 목업 |
|---|---|---|---|
| 1 | 매뉴얼 안에서만 답해요 | 근거 없는 추측은 하지 않고 "확인되지 않습니다"로 답함 — 신뢰의 기준 | 말풍선 + 근거 아이콘 |
| 2 | 물어보고 근거로 확인 | 컴포저에 질문 → 답변 + `[S1]` 인용 → 클릭 시 해당 근거로 이동 | 답변카드 + 인용칩 |
| 3 | 근거는 여기서 | 데스크톱=우측 근거 패널(지도·카드), **모바일=답변 하단 "근거 N건" 버튼**으로 시트 열림 | 3존/모바일 시트 대비 |
| 4 | 좁혀 묻고, 피드백 | "범위" 칩으로 부문 한정(다른 부문 혼입 방지) · 이상하면 「피드백」으로 알려주기 | 범위칩 + 피드백칩 |

마지막 스텝의 「시작하기」가 오버레이를 닫고 `pbdesk.onboarded=<버전>` 저장.

## 5. QA 모드 설명 (핵심) — 코치 + 도움말 탭

QA는 이 데모에서 가장 이해가 어려운 지점이므로 **한 줄 정의 + 항목별 평이한 표**로 설명한다.

> **QA 모드 = 검색 품질을 눈으로 확인하는 계측 모드.** 답을 얻는 데는 필요 없고, 검색이 왜 이렇게 나왔는지 뜯어볼 때(평가·튜닝) 켠다.

| 컨트롤 | 화면 라벨 | 쉬운 설명 |
|---|---|---|
| 혼합 α | `혼합 α` (BM25 키워드 ↔ Dense 의미) | 0 = 키워드(정확히 그 단어) 중심, 1 = 의미(비슷한 뜻) 중심, 0.5 = 반반 |
| 임계 τ | `임계 τ` | 관련도 통과선. 최고 점수가 τ보다 낮으면 "매뉴얼 밖"으로 처리. 낮추면 관대·높이면 엄격 **(답변 게이트는 다음 질문부터 적용)** |
| top-k | `top-k` | 근거로 가져올 청크 개수 |
| dense/sparse 막대 | 근거 카드 하단 | 이 근거가 **의미검색**·**키워드검색**에서 각각 얼마나 걸렸는지 |
| 게이트 모드 | `게이트: 코사인/리랭커` | 코사인=빠른 판정, 리랭커(정밀 토글)=느리지만 정확 |
| 유형 칩 | 화면개요·화면설명·용어찾기·관련화면·질문보기 | 근거 유형 필터(끄면 그 유형 제외) |

**진입 방법** 3가지를 코치 첫 화면에 명시: `Q` 키 · 헤더 `QA` 버튼 · 주소 `?qa=1`.
**코치 형태**: QA 패널이 실제로 열린 상태에서, 위 항목을 **순서대로 짧은 콜아웃 카드**로 넘겨 보여준다(패널 자체를 가리지 않게 하단/측면 배치). 마지막에 `pbdesk.qacoached` 저장. 데스크톱·모바일 모두 콜아웃은 화면 하단 고정 카드로 동일 처리.

## 6. 피드백으로 품질을 높이는 화면 — 설명

**프레이밍**: 이 데모의 답변 품질과 매뉴얼 커버리지는 **사용자 피드백으로 개선된다.** 온보딩·도움말은 "내 피드백이 실제로 답을 좋게 만든다"는 **품질 루프**를 보여줘 참여 동기를 만든다. QA 모드가 "품질을 *진단*"하는 창이라면, 피드백 화면은 "품질을 *개선*"하는 입구다 — 둘을 짝으로 설명한다. (기능 세부는 [사용자피드백_계획.md])

**왜 남기나 (품질 루프를 눈에 보이게)**

```
   내 지적 ─▶ ① 답변 품질  ─▶ 프롬프트·τ·검색 튜닝 + 회귀 평가 시드
            ├▶ ② 매뉴얼 최신화/보강 ─▶ 원문 매뉴얼 갱신 요청
            └▶ ③ 버그·제안 ─▶ 데모 개선
                    │
                    ▼
            반영되면 상태가 '반영'으로 → 보드·통계에서 내 눈으로 확인
```

- **어디서 남기나 (진입 4)** — 대부분 답변 옆에서 바로 열려 재현 맥락이 자동으로 붙는다.
  1. 답변 하단 **👍 / 👎** — 원클릭. 👎는 곧바로 상세 피드백으로 이어짐(질문·근거 자동 첨부)
  2. 답변 하단 **「피드백」** — 그 답변의 **질문·근거·게이트 점수가 자동 첨부**된 상태로 열림 → "답이 이상해요"가 아니라 조치 가능한 지적이 됨
  3. **게이트 차단**("매뉴얼에서 확인되지 않습니다") 답변의 **"보강 요청하기"** — 매뉴얼 공백을 가장 정확히 아는 순간
  4. 헤더 **「피드백」** 버튼 — 컨텍스트 없는 전역 등록 + 둘러보기 + 통계
- **무엇을 남기나 (유형 5)** — 버그 / 답변 품질 / 매뉴얼 최신화 / 매뉴얼 보강 / 제안·기타. 유형을 고르면 **작성 가이드·예시가 그 자리에서 바뀌어** 무엇을 적어야 조치 가능한지 알려줌.
- **어떻게 보나** — 「둘러보기」(유형 필터 · 공감 ♥ · 상태 배지)로 이미 올라온 의견을 확인·공감(중복 방지·참여감), 「통계」로 유형 분포·**처리 현황(반영률)**·답변 반응 추이를 투명하게 확인.
- **안심 안내** — 답변에서 열면 질문·근거가 함께 전송됨을 명시(무엇이 가는지 투명) + "개인정보·실계좌번호 금지" 문구.

미니 목업(온보딩·도움말 공용):

```
┌ 피드백 남기기 ─────────────┐   답변 하단:  👍  👎  [피드백]
│ [버그][품질][최신화][보강]  │
│ 💡 유형별 작성 가이드…       │   내 의견이 '반영'되면
│ [내용…] [📷 캡처] [등록]    │   보드에서 초록 배지로 확인
└──────────────────────────┘
```

**온보딩 배치**: 환영 캐러셀 §4 스텝4에서 "이상하면 알려주면 좋아져요"로 가볍게 씨앗만 뿌리고(안내 피로 최소화), **깊은 설명은 이 §6 + 도움말 「피드백」 탭**에 둔다. 답변을 처음 받은 직후 👍/👎 영역에 **1회성 말풍선 힌트**("도움이 됐는지 알려주시면 품질이 좋아져요")로 자연스럽게 유도(선택).

## 7. 상시 도움말 오버레이 — 탭 5개

헤더 「?」로 열리는 재열람 허브. 환영/코치/피드백 콘텐츠를 그대로 재사용(단일 출처).

- **기본 사용법** — §4 4스텝 요약
- **근거·범위** — 근거 지도 읽는 법, "좁히기"로 부문 한정, 정밀/빠른 검색 차이
- **QA 모드** — §5 표 전체
- **피드백으로 품질 높이기** — §6 전체(품질 루프 · 진입 4 · 유형 5 · 반영 확인). QA 탭과 "진단↔개선" 짝으로 안내
- **단축키·모바일** — `/`·`⌘K` 입력 포커스, `Q` QA 토글, `Esc` 닫기 / 모바일 ☰ 이력·"근거 N건" 시트·QA·피드백 위치

## 8. 모바일 대응

- 오버레이는 **풀스크린 시트**(피드백 모달의 `@media(max-width:560px)` 패턴 확장), 큰 탭 타깃, 스와이프 + 점 인디케이터.
- **모바일 전용 안내 문구**를 명시: 근거는 답변 하단 **"근거 N건"** 버튼, 이력은 좌상단 **☰**, QA는 헤더 **QA** 토글, 피드백은 헤더 **「피드백」**·답변 하단 **👍/👎/피드백**. (데스크톱과 위치가 다른 항목)
- QA 슬라이더(α·τ)는 터치로 조작 가능함을 코치에서 한 줄 안내.
- 본문 가로 스크롤 금지, 목업 SVG는 `max-width:100%`.

## 9. 데이터·영속 (localStorage, 서버 무관)

```
pbdesk.onboarded = "ob1"     환영 완료(가이드 버전). 버전 상수 OB_VER 로 관리 — 개편 시 올리면 재노출
pbdesk.qacoached = "qa1"     QA 코치 완료
```

- 순수 프런트 추가 — `index.html`·`app.js`·`styles.css`만 수정, API·데이터 무변경.
- `?onboarding=1` 파라미터로 강제 표시(QA·데모 시연용).

## 10. 구현 단계

### 1단계 — 도움말 허브 (재열람 기반부터)
헤더 「?」 버튼 + 탭형 도움말 오버레이(§7) + QA 탭(§5) + 피드백 탭(§6) 콘텐츠. 콘텐츠를 상수(`GUIDE`)로 단일화해 이후 단계가 재사용.

### 2단계 — 최초 방문 환영 캐러셀
§4 4스텝 + 빈 히어로 "사용법 보기" 링크 + `onboarded` 플래그. 스와이프/점/키보드 네비.

### 3단계 — QA 첫 활성화 코치
`setQa(true)` 최초 호출 시 `qacoached` 없으면 콜아웃 시퀀스(§5) + 플래그. 진입 3방법 안내.

### (선택) 4단계 — 사내 webapp 이식
동일 컴포넌트를 `web/`에 이식(QA 패널 구조 동일하므로 콘텐츠 거의 무수정).

## 11. 검증

- **최초 방문** → 환영 자동 노출 → 닫으면 재방문 시 미노출, 「?」로 재열람 가능.
- **QA 첫 진입**(`Q`) → 코치 1회 → 이후 미노출. `?qa=1` 딥링크로도 코치 동작.
- **피드백 설명** → 도움말 「피드백으로 품질 높이기」 탭이 품질 루프·진입 4·유형 5·반영 확인을 담고, 첫 답변 후 👍/👎 1회성 힌트가 표시되는지(선택 구현 시).
- **모바일**(≤640) → 스와이프·풀스크린 시트·슬라이더 안내 정상, **가로 스크롤 0**, "근거 N건/☰/QA" 위치 안내 정확.
- **접근성** → `Esc` 닫기, 포커스 트랩, `prefers-reduced-motion`에서 애니메이션 정지.
- **회귀** → 기존 질의/근거/피드백 흐름 무영향(프런트 순수 추가). `?onboarding=1`로 강제 표시해 반복 확인.
- 배포 후 프로덕션에서 신규 시크릿창으로 최초 노출 확인 + localStorage 리셋 재확인.
