## 2. 목표 / 범위

기존 화면·업무매뉴얼과 **함께 검색**되는 제3의 매뉴얼로 "상담"을 추가한다.

- 청킹은 **별도 로직**(Q&A 쌍 기반 — HTML 파서와 무관한 신규 모듈).
- `고객사`·`비고` 열도 **보존** — 고객사는 사례 맥락(어느 회원사 사례인지), 비고는 보충 메모로 검색·표시 모두에 살린다.
- 스코프는 기존 TOC처럼 **1-level**: `상담 > {업무명 19종}` — 셀렉터·필터·근거 지도가 무수정으로 동작하도록 기존 `sector_path` 루트 설계를 그대로 탄다.
- 범위: Upstash 업서트 + `_static.py`(sectors/meta) 갱신 + 프런트 소폭. **로컬 FAISS 색인·webapp은 범위 밖.**

## 3. 청킹 설계 (신규 모듈, 별도 로직)

**단위: Q&A 쌍 1개 = 청크 1개** (상담 데이터의 자연 단위 — 브레드크럼 대응물).

행 순회 규칙(실사 기반):
1. 헤더 행 스킵 — `질문내용` 열 값이 "질문내용"(공백 제거 비교)인 행.
2. **새 쌍 시작** — 질문내용 열이 비어있지 않은 행. 고객사·비고 열도 함께 수집(고객사명 내 개행 정리: "리딩투자⏎증권"→"리딩투자증권").
3. **연속 행 병합** — 질문내용이 빈 행의 답변 텍스트는 직전 쌍의 답변에 `\n` 이어붙임(질문·비고 연속도 동일 규칙으로 각 필드에 병합).
4. 정규화 — 셀 내 개행 유지·NBSP/다중공백 정리(기존 `parse.norm` 규칙 이식), float 셀(숫자) 문자열화.
5. **빈 쌍 제거** — 질문 또는 답변이 5자 미만이면 폐기(로그 집계).
6. **중복 병합** — 동일 Q&A가 복수 고객사로 반복 등장 가능 → `sha1(정규화(질문+답변))` 완전 일치 시 청크 1개로 합치되 **고객사 목록을 병합**(`고객사: A, B`)·비고도 상이하면 이어붙임(로그 집계). 질문 같고 답변 다른 쌍은 모두 유지.
7. **PII 마스킹(2026-07-09 스캔 후 추가, 사용자 승인)** — 고객사·담당자명은 보존하되(§1 결정 유지) 3종만 보완: ①카드번호 풀노출→`5531-****-****-****` ②무마스킹 실계좌번호→원본 문화와 같은 끝자리 `***` 방식(테스트 계좌 `999-…`·`001-00-0000xx`·전자리 동일은 제외) ③마스킹 미흡 식별번호(7자리+ 숫자 뒤 `*`)→앞 4자리만. 단일 `*` 곱셈·소수점 산식은 제외 규칙으로 보호(`parse_counsel_xls.py mask_pii`). 영향 28청크만 동일 id 재업서트로 반영.

## 4. 청크 스키마 매핑 (기존 온라인 메타와 정합)

`ingest_real.py`의 업서트 메타·`write_static` 트리 빌더가 그대로 소비할 수 있는 형태로 산출:

| 필드 | 값 | 근거 |
|---|---|---|
| `id` | `cs:{업무slug}:{0001}` | 기존 `pm:` 접두 관례의 상담판 |
| `manual` | `"상담"` | `scope_hint`가 `h.get("manual")`을 우선 사용 → 3매뉴얼 모호성 배너 무수정 동작 |
| `sector_path` | `["상담", 업무명]` | **1-level** — 셀렉터 루트에 "상담" 자동 등장, `scope=상담>계좌` GLOB 필터 무수정 동작 |
| `sector` | 업무명 (예: `계좌`) | 부문 분포·배너용 |
| `screen_id` / `title` | `CS01`~`CS19` / `"{업무} 상담 Q&A"` | `write_static` 트리의 문서 레벨 요구 충족(업무당 1문서) |
| `screen_no` / `source_url` | `""` / `""` | 화면번호 없음 — UI는 빈 값 표기 생략(기존 관례). 원문 링크는 프런트에서 조건부 처리(§6) |

| `section_path` | `["상담사례", 질문(120자 절단)]` | 근거 카드 경로에 질문이 보이게 |
| `text` | `"Q. {질문}\nA. {답변}"` + 말미 `"― 고객사: {A, B}"`·`"― 비고: {…}"`(있을 때만) | 카드 본문·추출형 답변에서 쌍+맥락이 온전히 보이게. **고객사·비고를 text에 접는 방식이라 ingest 메타 스키마 무변경** |

| `term` | `""` | — |

## 5. 파이프라인 (스크립트 2개)

### ① `deploy/online/parse_counsel_xls.py` (신설, 로컬 실행 도구 — 배포 제외)

```bash
uv pip install --python .venv/bin/python xlrd     # 이미 설치 확인(2.0.2) — requirements-tools 주석으로만 기록
python deploy/online/parse_counsel_xls.py         # → data/chunks_counsel.jsonl + 통계 출력
python deploy/online/parse_counsel_xls.py --stats # 파싱만(파일별 쌍/병합/중복/마스킹 건수 표)
```

§3 규칙으로 19개 파일 → `data/chunks_counsel.jsonl`. 파일별 통계(행→쌍→중복제거→최종)를 stdout 표로 — 수지 검증의 근거.

### ② `ingest_real.py` 최소 확장 (기존 재개 로직 재사용)

`load_chunks()`가 `chunks.jsonl` + `chunks_counsel.jsonl`(있으면)을 **이 순서로 연결**해 읽도록 한 줄 확장.

- **증분 업로드가 공짜로 해결**: 진행 오프셋(`.ingest_state`)이 기존 44,330에 멈춰 있으므로, 병합 리스트 뒤에 상담 청크가 붙으면 다음 `--limit` 실행이 **상담분만** 이어서 업서트(기존 재업서트 없음). 전제: `chunks.jsonl`은 그동안 불변.
- `write_static()`은 병합 리스트로 재생성 → sectors 트리에 `상담` 루트(하위 업무 19개) 자동 반영, `meta.count`·부문 분포 갱신.
- `--reset` 불필요(추가 업서트).

## 6. 프런트 최소 수정 (`public/` 3곳)

1. **근거 카드 원문 링크 조건부** — `source_url` 빈 청크(상담)는 "원문 매뉴얼 ↗" 링크 미표시(현재는 빈 href 렌더됨).
2. **매뉴얼 배지** — 현재 `업무`만 배지 표시 → `상담` 배지 추가(색 구분, 기존 `.m-badge` 변형).
3. **온보딩·도움말 문구** — "화면·업무" 언급부에 "상담사례" 반영(도움말 근거·범위 탭 1문장).

무수정 동작 확인 대상(설계 근거): 스코프 셀렉터(`/api/sectors` 트리 루트 3개), `scope=상담>…` 필터, 매뉴얼 모호성 배너(3파전), 부문 배너, 근거 지도(상담 루트 노드), 피드백 ctx.

## 7. 단계별 계획

### 1단계 — 파서 + 검증 (로컬)
`parse_counsel_xls.py` 구현 → 전 파일 파싱 통계 + 골든 스모크(예: `계좌` 첫 쌍/마지막 쌍 텍스트, 파일별 쌍 수 스냅샷을 스크립트 내 assert로). 고객사 병합 샘플(복수 고객사 청크 상위 10건)·비고 보존 샘플 출력으로 눈검증.

### 2단계 — 업서트 + 정적 갱신 + 프런트
`ingest_real.py` 확장 → 상담분 업서트(예상 ~8.5K, 1일 내) → `_static.py` 재생성 → 프런트 3수정 → `vercel deploy --prod`.

### 3단계 — 온라인 검증 + 튜닝
- 스코프: `?scope=상담`·`상담>계좌`에서 상담만 히트, 역으로 `화면>계좌` 질의에 상담 미혼입.
- 게이트: 상담 질문 몇 개로 τ(0.70) 유지 적정성 확인(Q&A 문체가 달라 코사인 분포가 다를 수 있음 — 필요 시 GATE_TAU 재보정).
- 모호성 배너: "계좌 비밀번호" 류 질의에서 화면/업무/상담 3-매뉴얼 배너 동작.

### 4단계(선택) — 질문뱅크 연계
상담 질문은 천연 추천 질문 재료 — `gen_questions.py`에 상담 소스 추가(자기-검색 검증 통과분만, 부문 라운드로빈에 `상담` 그룹 합류).

## 8. 검증 방법

- **수지 검증**: 파일별 `행수 → Q시작행 → 병합후 쌍 → 중복병합 → 최종청크` 표가 실측(총 9,193 Q행)과 정합.
- **골든 스모크**: 대표 3파일(계좌/출납/회계)의 첫·끝 쌍 질문 원문 assert. 고객사·비고가 있는 원본 행 표본 10건이 청크 text에 그대로 보존됐는지 대조.
- **온라인**: `/api/meta` count 증가분 = 최종청크 수, `/api/sectors` 루트에 상담(하위 19), §7-3단계 시나리오, 근거 카드에 원문 링크 없음·상담 배지 표시. "○○증권 ○○ 사례" 류 고객사 포함 질의가 해당 상담 청크를 히트하는지 확인(embed_text 포함 효과).
- **회귀**: 기존 화면·업무 질의 3종의 top-hit 불변(추가 업서트는 기존 벡터 무변경이므로 순위 변동은 신규 혼입에 의한 것만).
