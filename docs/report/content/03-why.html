Why — 배경

매뉴얼은 있는데, 찾기가 일이었습니다

원장 매뉴얼은 수백 개 화면이 트리 메뉴에 흩어져 있어, 필요한 항목 하나를 찾으려면 메뉴를 열고 닫으며 화면을 하나하나 확인해야 했습니다. 화면 이름을 모르면 검색조차 어렵고, 그 사이 같은 질문이 담당자에게 매일 반복됩니다.

일반적인 해법인 상용 AI 서비스는 폐쇄망·데이터 보호 요건상 사용할 수 없습니다. 원장 매뉴얼은 외부로 한 글자도 나갈 수 없기 때문입니다. 그래서 사내 서버 안에서 완결되는 자체 구축을 선택했습니다.

기존 방식의 비용
탐색 반복 — 트리 메뉴 클릭·스크롤의 연속
진입 장벽 — 화면 이름을 알아야 찾는 구조
반복 문의 — 담당자의 시간 잠식
제약 — 상용 AI API 사용 불가(데이터 반출 금지)
기존 — 트리 메뉴 탐색의 반복 vs 이제 — 질문 한 줄 (소개 영상 중)
