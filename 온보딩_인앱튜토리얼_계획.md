# 온보딩 강화 — 인앱 튜토리얼(하이라이트/스포트라이트 코치마크) 계획 v2

2026-07-09. 대상: 온라인 데모(`deploy/online/`) 우선 → 사내 `web/` 이식. 순수 프런트(app.js·styles.css·index.html), 서버·API 무변경. 참고: driver.js/intro.js 개념(devlab.tistory.com/3) — 무의존 제약으로 **라이브러리 미사용, 바닐라 구현**.

## 1. 방침 변경 (v2)

- **데스크톱(PC) 전용으로 최적화**. 모바일은 스포트라이트 투어를 **띄우지 않고** 기존 캐러셀/도움말만 유지(모바일에선 근거·이력이 슬라이드 시트라 하이라이트 정합이 애매 — 무리하지 않는다). 투어 진입점은 `min-width:960px`에서만 노출. 좁은 화면에서 hero-guide는 기존 캐러셀을 연다.
- **훨씬 친절하고 상세하게**. PC에선 3존(이력·대화·근거)이 항상 보이므로, 레이아웃 오리엔테이션 → 컴포저 조작 → **실제 예시 시연**까지 이어지는 다-챕터 가이드로 만든다. 각 스텝은 한 문장이 아니라 "무엇을·왜·어떻게(+팁)"로 서술.

## 2. 설계 원칙

- **무의존 바닐라 스포트라이트**: `box-shadow: 0 0 0 9999px rgba(dim)` 기법으로 대상 하나만 밝히는 구멍(cutout) — SVG 마스크 불필요. `getBoundingClientRect`로 위치, `scroll`/`resize`/`ResizeObserver`에 rAF 디바운스 재배치. 데스크톱 전용이라 시트 prep/cleanup 복잡성 없음.
- **부재/숨은 대상 자동 스킵**: 대상이 없거나 `hidden`/`offsetParent===null`이면 그 스텝 건너뜀(방향 유지). 예: `#precise`는 온라인 숨김 → 자동 스킵, 사내판엔 등장.
- **강요하지 않되 친절**: 스텝 수가 늘어도 항상 「건너뛰기」·`Esc`, 상단 "n/총계 · 챕터명" 진행 표시, 「이전」 자유 이동. 문구는 혜택 우선.
- **실물 위주**: 추상 목업 대신 실제 요소를 하이라이트. 근거·인용처럼 답변 후 생기는 요소는 **가이드 시연(예시 질문 자동 실행)**으로 실제 콘텐츠를 만들어 그 위에 하이라이트 → 개념이 아니라 "방금 일어난 일"로 이해.
- **기존 자산 공존**: 캐러셀·탭 도움말·QA 코치 유지. 투어는 "화면에서 자세히 둘러보기" 레이어로 얹고 진입점 연결.
- **접근성**: 툴팁 `role="dialog"`+포커스 트랩, `←/→` 이전·다음, `Esc` 종료, `Enter` 다음, 스텝 문구 `aria-live`. `prefers-reduced-motion` 시 스포트라이트 이동 트랜지션 제거. 큰 탭 타깃.
- **버전드 완료 플래그**: `pbdesk.tour`(기존 `pbdesk.onboarded`와 별개) — 개편 시 재노출.

## 3. 스포트라이트 엔진 (app.js 온보딩 블록 ≈1414~에 신규)

```
TOUR = { steps, i, active }
startTour(steps, from): 데스크톱 아니면 캐러셀로 폴백. .tour-scrim/.tour-hole/.tour-tip 1회 생성, i=0
renderStep():
  s=steps[i]; el=resolve(s.target)                 // 없음/hidden → 방향대로 skip
  await s.before?.()                                // 예: 시연 스텝은 예시 질문 실행·답변 대기
  requestAnimationFrame(() => {
    r=el.getBoundingClientRect()
    hole ← r + pad(s.pad ?? 8), border-radius 매칭, box-shadow 스포트라이트, 옵션 펄스 링
    tip  ← r 기준 placement 자동 flip(상하좌우 공간·경계 감지), 제목·본문(HTML)·진행·이전/다음/건너뛰기
    필요 시 el.scrollIntoView(뷰포트 밖일 때만; reduced-motion이면 즉시)
    다음 버튼 포커스
  })
end(completed): DOM 제거, completed면 pbdesk.tour=버전
재배치: scroll/resize/ResizeObserver → 위치만 rAF 갱신.  키보드: ←/→/Esc/Enter. 스크림 클릭=다음.
```

- z-index 90대(scrim 90/hole 91/tip 92) — 기존 스크림 40·시트 50·fb 60·라이트박스 70 위 최상단.
- 하이라이트 구멍 기본 `pointer-events:none`+뒤 요소 클릭 차단(툴팁 버튼으로만 진행, 오조작 방지). 단 시연 스텝은 예외적으로 실제 인용 클릭 허용 옵션.

## 4. 투어 시나리오 — 3 챕터 (상세·친절, 데스크톱)

각 스텝: **제목 / 왜(혜택) / 어떻게 / 💡팁**. 진행 표시 "챕터 · n / N".

**챕터 1 — 화면이 이렇게 생겼어요 (레이아웃 오리엔테이션)**
1. `.shell`(또는 순차) 전체 — "왼쪽 이력 · 가운데 대화 · 오른쪽 근거, 세 칸으로 나뉘어요."
2. `#rail`(좌) — "질문할 때마다 여기에 이력이 쌓여 다시 열어볼 수 있어요."
3. `#evidence`(우) — "답의 출처가 지도·카드로 여기 나타나요. 답을 믿을 근거를 늘 함께 보여줘요."
4. 컴포저 `#composer` — "질문은 여기서. 지금부터 하나씩 알려드릴게요."

**챕터 2 — 이렇게 물어보세요 (컴포저 조작)**
5. `#q` 입력창 — 무엇/어떻게 + 💡 `Enter` 전송·`Shift+Enter` 줄바꿈, `/`·`⌘K`로 포커스.
6. `#scope-chip` 범위 — "부문을 좁히면 다른 업무 결과가 섞이지 않아요." + 💡 답이 여러 부문에 걸치면 배너로 되물어봐요.
7. `#samples`(추천 칩) — "뭘 물을지 막막하면 검증된 추천 질문으로 시작해요. ↻로 새로고침."
8. `#precise` — (표시될 때만) 정밀/빠른 차이. 온라인 숨김이면 자동 스킵.

**챕터 3 — 실제로 해볼게요 (가이드 시연)**
9. 시연 안내 카드 — "예시로 한 번 보여드릴까요?" [예시 보기] / [직접 할게요]. 후자는 챕터 종료.
10. `before`: 예시 질문(추천 칩 1개 또는 상수)을 `ask()`로 자동 실행 → `done` 대기. 실패/게이트면 다른 예시 1회 재시도, 그래도 실패면 시연 스킵하고 안내.
11. 답변 첫 `.cite`([S1]) 하이라이트 — "문장 끝 [S1]을 누르면 근거가 된 매뉴얼 위치로 이동해요." (여기서만 실제 클릭 허용)
12. `#ev-map`/첫 `.ev-card` — "근거 지도·카드에서 어느 화면·부문에서 나온 답인지 확인해요. 화면번호로 단말에서 바로 찾을 수 있어요."
13. 마무리 — `#help-open`·`#fb-open`·`#qa-toggle` 순차(또는 한 컷) — "사용법은 ?에서 다시, 이상하면 피드백으로. 품질을 뜯어보려면 QA." + 「둘러보기 완료」.

※ 총 ~10~12스텝이지만 진행 표시·자유 이동·상시 건너뛰기로 부담 완화. "빠른 3스텝" 대신 상시 스킵으로 길이 문제 해소(사용자 요구=상세함 우선).

## 5. 진입점 연결

- 환영 캐러셀 마지막: 「시작하기」 옆 **「화면에서 자세히 둘러보기」** → `startTour`.
- hero-guide("처음이신가요?")·`?` 도움말 '기본' 탭: 「화면 둘러보기」 버튼 상시.
- 재방문자(`pbdesk.onboarded` 있고 `pbdesk.tour` 없음, 데스크톱): hero-guide 1회 펄스로 발견성만 ↑(자동 시작 안 함).
- 모바일: 위 진입점은 캐러셀로 폴백(투어 미노출).
