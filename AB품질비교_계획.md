# A/B 품질비교 평가 계획 — PB 매뉴얼 데스크 vs PB AX (PoC)

2026-07-10. 동일 질문셋으로 두 매뉴얼 챗봇의 품질을 블라인드·정량 비교한다.

## 1. 대상 시스템 (2026-07-10 실사)

| | **A — PB 매뉴얼 데스크** (우리) | **B — KOSCOM PB AX (PoC)** |
|---|---|---|
| 주소 | pb-manual-desk-demo.vercel.app | 133.186.250.105:12102/reports/manual-search |
| API | `GET /api/answer?q=…` (JSON) | `POST /reports/manual-search/chat` `{query, sessionId, stream:false, domains}` (JSON) — **실호출 검증 완료** |
| 검색 | Upstash 하이브리드(dense text-embedding-3-small + BM25, DBSF) + 코사인 게이트 τ0.70 | 자체 RAG(패시지 검색 + query rewrite) |
| 답변 LLM | OpenAI gpt-5.4 | gemma4-31b |
| 인용 | `[S1]` 마커 + `hits[].source_url` | `[1]` 마커 + `references[].info.link` |
| 커버리지 | **3매뉴얼** — 화면 39,240 · 업무 5,100 · 상담 9,123 청크 | **2도메인** — 화면(powerbase_help) · 업무(powerbase_manual). 상담 없음 |
| 거부 동작 | 게이트 미달 시 "매뉴얼에서 확인되지 않습니다" | (평가로 확인) |

**결정적 공통점**: 두 시스템 모두 근거가 **동일한 사내 RoboHelp 원문**(`211.255.203.234:8080/...#t=XXXX.html`)을 가리킨다 → 근거 문서 일치를 **파일명 정규화로 자동 채점** 가능(예: A `source_url` vs B `references[].info.link` 모두 `ACM20700.html`).

## 2. 공정성 설계 (객관성 장치)

1. **as-served 비교** — 각 시스템 운영 기본값 그대로(A: 전체 스코프·기본 topk, B: 두 도메인 모두 on, systemPrompt 빈 값). 어느 쪽도 이 평가를 위해 튜닝하지 않는다.
2. **본평가는 공통 커버리지(화면+업무)만** — 상담 질문은 A만 데이터가 있어 불공정 → 별도 "커버리지 트랙"으로 분리 보고.
3. **세션 오염 차단** — B는 `sessionId`를 질문마다 새로 발급(query_rewrite가 대화 이력을 참조하므로), A는 stateless.
4. **인터리브 실행** — A→B, B→A를 교대로 같은 시간대에 실행(서버 부하 시점 편향 제거), 문항 간 2초 지연(B PoC 보호).
5. **블라인드 심판** — 심판에게 시스템 정체 숨김(응답을 "시스템 갑/을"로 제시), **문항마다 갑/을 배치 무작위 스왑**(포지션 바이어스 제거), 마커 표기(`[S1]`/`[1]`)는 스타일 단서라 심판 입력 전에 `[근거]`로 통일 정규화.
6. **이중 심판 + 사람 검수** — LLM 심판 2종(claude CLI + gpt-5.4, 서로 다른 벤더)의 독립 채점 → 심판 간 일치도(κ) 보고, 불일치 문항과 표본 20%는 사람(사용자) 최종 판정.
7. **원자료 전량 보존** — 두 시스템의 raw 응답 JSON을 그대로 저장(재채점·이의 제기 가능).

## 3. 질문셋 — 60문항 (gold 기준 포함)

| 트랙 | 문항 | 구성 방법 | gold(정답 기준) |
|---|---|---|---|
| T1 검증 질문뱅크 | 30 (화면 20·업무 10) | `api/_questions.py` 881건(자기-검색 검증済)에서 부문 층화 무작위 샘플 | 출처 화면 sid → 원문 html 파일명 |
| T2 난이도 상향 | 10 | 수기 작성 — 절차형(단계 나열)·비교형(두 화면 차이)·조건형(예외 상황) | 원문 대조로 정답 요지 수기 확정 |
| T3 함정/거부(OOS) | 10 | 매뉴얼 밖 질문 5(일상·타사 시스템) + 존재하지 않는 화면번호/기능 5 | "모른다/확인 불가"가 정답 |
| T4 커버리지(상담) | 10 | 상담 Q&A 원문에서 샘플(별도 트랙 — B 불리 명시) | 상담 A. 원문 |

- T1은 질문뱅크가 이미 화면별 출처를 알고 있어 **retrieval 채점이 완전 자동**.
- 질문셋은 실행 전 확정·동결(`data/ab_eval/questions.jsonl`)하고 이후 수정 금지.

## 4. 지표

| 지표 | 정의 | 채점 방식 |
|---|---|---|
| ① 근거 적중률 | gold 문서가 top-3 근거(A hits / B references)에 포함 | **자동** (파일명 정규화 매칭) |
| ② 답변 정확성 | 1~5 루브릭(5=원문과 완전 일치, 3=부분 정답/누락, 1=오답) | 블라인드 LLM 이중 심판 — 원문 gold 첨부해 대조 채점 |
| ③ 근거 충실성 | 답변 주장 중 제시 근거에 없는 내용(환각) 존재 여부 | LLM 심판(근거 원문 대조) + 사람 표본 |
| ④ 거부 적절성 | T3에서 정직한 거부율 — 그럴듯한 오답 생성 = 실패 | 자동(거부 문구) + 사람 확정 |
| ⑤ 인용 품질 | 인용 마커 존재·마커→근거 대응 정확·원문 링크 유효 | 자동 + 표본 |
| ⑥ 응답 속도 | end-to-end p50/p95 (ms) | 자동(하네스 측정) |
| ⑦ 쌍대 선호 | 문항별 "어느 답이 실무자에게 유용한가" 3지선다(갑/을/무승부) | 블라인드 이중 심판 |
| ⑧ 커버리지 | T4 답변 가능률(정답 포함률) | ② 루브릭 재사용, 별도 표 |

종합은 단일 점수로 뭉개지 않고 **지표별 표 + 트랙별 승패 + 대표 사례**(양쪽이 갈린 문항 5개 원문 인용)로 보고 — 강점 프로파일이 다른 시스템을 한 숫자로 줄이면 왜곡된다.

## 5. 실행 하네스 (전부 로컬 스크립트, 저장소 오염 없음)

`deploy/online/ab_eval/`(로컬 실행 도구, 배포 제외):

1. `make_questions.py` — §3 질문셋 생성(T1 자동 샘플 + T2·T3 수기 목록 병합) → `questions.jsonl` 동결.
2. `run_ab.py` — 문항별 A(`/api/answer`)·B(`/chat`) 인터리브 호출, raw 응답·레이턴시 → `runs/{a,b}/{qid}.json`. B 세션 만료 시 중단·안내(사용자 재로그인 필요).
3. `score_auto.py` — ①⑤⑥ 자동 채점 + ④ 1차 판정 → `scores_auto.csv`.
4. `judge_blind.py` — ②③⑦ 심판 입력 패킷 생성(정체 은닉·마커 정규화·순서 스왑 기록) → claude CLI·OpenAI 이중 채점 → `scores_judge.csv` + κ.
5. `report.py` — 지표 표·트랙별 요약·사례 발췌 → `AB품질비교_보고서.md`.

예상 소요: 하네스 구현 반나절, 실행 60문항×2시스템 ≈ 20분(레이턴시 포함), 심판 ≈ 10분, 사람 검수 20~30분.

## 6. 사람 개입 지점 (사용자)

- 질문셋 동결 전 T2·T3 문항 승인(10+10개, 한 화면 분량).
- 심판 불일치 문항 + 20% 표본 최종 판정(예상 15~20문항).
- 보고서 결론 검토.

## 7. 리스크·주의

| 항목 | 대응 |
|---|---|
| B 세션 만료(sessionToken) | 하네스가 401/로그인 리다이렉트 감지 시 즉시 중단·재로그인 요청. 실행 직전 프로브 1회 |
| B PoC 부하/불안정 | 문항 간 2초 지연, 타임아웃 90s, 실패 문항 1회 재시도 후 "무응답"으로 기록(0점 아님 — 별도 표기) |
| 심판 편향 | 이중 심판(벤더 교차)·블라인드·스왑·루브릭 강제·판정 이유 필수. self-preference 주의: A가 GPT계열이므로 OpenAI 심판 단독 사용 금지 |
| 상담 트랙 해석 | "B에 없는 데이터" 명시 — 시스템 우열이 아니라 데이터 커버리지 차이로 보고 |
| 평가 중 시스템 변경 | 실행 당일 A·B 둘 다 변경 동결(우리 쪽 배포 금지 포함) |
| B 응답 스타일 차이(어미 "~한다") | 품질 판정에 문체 불이익 금지를 루브릭에 명시(정확성·근거 중심) |
